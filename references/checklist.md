# Чеклист готовности к запуску сборки

Перед вставкой промпта в Claude Code пройди этот чеклист. Каждый пропущенный пункт — часы на отладку.

## Документ идеи (PROJECT_IDEA.md)
- [ ] Проблема с числами и примерами
- [ ] Решение — пошаговый процесс, каждый шаг = модуль
- [ ] Архитектура с диаграммой слоёв
- [ ] Стек с обоснованием выбора
- [ ] Монетизация с планами и ценами
- [ ] Конкуренты в таблице
- [ ] MVP выделен отдельно от v2

## Техническая спецификация
- [ ] Каждый модуль: user stories + данные + API + UI + логика + edge cases
- [ ] Типы полей как в SQL (uuid, text, jsonb, timestamptz)
- [ ] API: метод + путь + тело + ответ + коды ошибок
- [ ] RLS для каждой таблицы
- [ ] AI-агенты описаны (если есть): модель, промпт, tools
- [ ] Нет TODO-заглушек — всё конкретно

## Конфигурация (Генератор)
- [ ] CLAUDE.md до 120 строк
- [ ] Модели правильно выбраны: opus для сложного, sonnet для рутины
- [ ] qa-reviewer без Write/Edit (только Read, Bash, Glob, Grep)
- [ ] Rules с glob-паттернами привязаны к нужным файлам
- [ ] MCP: Context7 + Supabase подключены
- [ ] Финальный промпт содержит ВСЕ файлы с точным содержимым

## Перед запуском Claude Code
- [ ] Репозиторий инициализирован (git init + первый коммит)
- [ ] Supabase-проект подключён (supabase link)
- [ ] Переменные окружения настроены (.env.local)
- [ ] Claude Code установлен и авторизован
- [ ] Файловая структура создана (.claude/agents/, .claude/rules/, .claude/skills/)

## Быстрая самопроверка перед генерацией

Задай себе эти вопросы:

1. **Могу ли я объяснить каждый модуль за 30 секунд?** Если нет — недостаточно описан.
2. **Есть ли в спецификации хоть одно "продвинутый/умный/гибкий"?** Замени на конкретику.
3. **Знает ли субагент, что делать, если что-то пошло не так?** Крайние случаи покрыты?
4. **Не превышает ли CLAUDE.md 120 строк?** Если да — вынеси детали в rules/skills.
5. **Все ли субагенты имеют правильные инструменты?** qa-reviewer без Write/Edit?
